# Gerar os arquivos de PRD

Precisamos criar os arquivos de PRDs no diretorio `docs/prds/`, esses arquivos devem começar com o nome `PRD-0XX-implementacao-playwright-{resumo}`, onde `0XX` é o número do PRD (ex: 001, 002, etc) e `{resumo}` é um resumo curto do conteúdo do PRD (ex: "testes-e2e", "integração-playwright", etc).

O primeiro PRD vai ser 019.

O conteúdo do PRD deve seguir o PRD-018-integracao-Docling.md.

O conteudo do PRD deve ter as informacoes suficientes para um desenvolvedor/agent de IA implementar a solução, incluindo detalhes técnicos, estrutura de arquivos, e integração com o pipeline existente.

Indique os ADRs do projeto que estao no diretorio `docs/adr/`.

## Base para os PRDs

Os PRDs vao conduzir a implementação do Discovery da integração do Playwright para testes E2E descrito em inicio-notas.md.

Os casos de uso estao mapeados em casos-de-uso.md, e a estrutura de arquivos proposta esta em casos_de_uso_e2e.md

Observacao: avalie se temos que ter 1 PRD para cada caso de uso, ou se podemos agrupar varios casos de uso relacionados em um unico PRD. Faca essa analise com base no uso de tokens para realizar a tarefa e evitar alucinações, e na clareza da implementação.

## O que nao devemos fazer

- ja criar os arquivos de PRD antes de realizar validar comigo quais PRDs serao criados com seus titulos e resumos, para evitar retrabalho.

- PRDs muito genéricos que nao tem detalhes suficientes para a implementacao, ou PRDs muito especificos que poderiam ser agrupados.

- PRDs muito grandes que tentam cobrir muitos casos de uso diferentes, o que pode dificultar a implementacao e a revisao.

## Você deve:

- realizar peguntar a mim, para garantir que os PRDs criados estejam alinhados com o que eu espero e com as necessidades do projeto.

- apos eu aprovar os PRDs, criar os arquivos de PRD seguindo a estrutura e o conteudo definido, garantindo que cada PRD tenha informacoes claras e detalhadas para a implementacao.

- depois de criar os arquivos de PRD gerar o prompt para eu passar para o agente de IA que vai realizar a implementacao, garantindo que o prompt seja claro e contenha todas as informacoes necessarias para a tarefa, o prompt deve indicar que eh um plano, onde cada PRD deve ser feito individualmente, e que o agente de IA so deve avancar para o proximo PRD apos a conclusao do PRD atual.
