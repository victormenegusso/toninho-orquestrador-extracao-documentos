# Organização do Projeto (discovery)

## Contexto

O projeto toninho, foi ate o momento criado utilizando linguagem e ferramentas que eu não tinha familiaridade, tambem eu estava testando formas de trabalhar com prompts, logo alguns pontos de organização do projeto foram deixados de lado, e outros foram criados de forma improvisada, o que fez com que o projeto ficasse um pouco bagunçado, e dificil de entender para outras pessoas. Tambem muito do codigo pode nao estar seguindo as melhores praticas.

## Objetivo

O objetivo é organizar o projeto de forma clara e estruturada, para que outras pessoas possam entender facilmente o que foi feito, e como o projeto funciona. Tambem é importante para mim, para que eu possa ter uma visão geral do projeto, e identificar pontos de melhoria. Precisamos garantir que a organização garanta que agents de IA possam entender o projeto e contribuir para ele, sem a necessidade de intervenção humana.

Para isso, vamos gerar um arquivo MARKDOWN de discovery, que sera usado para criar arquivos de PRD, para na sequencia atuarmos nas correcoes.

### Minha dor atual

A desorganização do projeto, que dificulta a compreensão do que foi feito, e como o projeto funciona. Tambem dificulta a identificação de pontos de melhoria. Tambem tem documentação que pode estar desatualizada, ou incompleta, o que dificulta ainda mais a compreensão do projeto.

Hoje o processo de execucao pelo docker, muitas vezes o docker demora para subir, devido a instalação de dependências.

O frontend muitas vezes bugs pequenos voltam a ocorrer.
